# 🌤 Weather Calendar Web App 設計書（Vercel前提）

---

## 🎯 目的

ユーザーが設定した地域の**直近1週間の天気予報**を、**カレンダーUI上に表示するWebアプリ**を構築する。

---

## 🏗 全体アーキテクチャ

```
[ React (Frontend / Vercel) ]
            ↓
[ API Routes (Vercel Serverless Functions) ]
            ↓
[ Python API (FastAPI / 外部 or Vercel Python Runtime) ]
            ↓
[ Open-Meteo API ]
```

---

## ⚙ 技術スタック

### Frontend

* React (TypeScript)
* FullCalendar
* Axios / fetch

### Backend

* FastAPI（Python）
* または Vercel Serverless Functions（Node or Python）

### External API

* Open-Meteo
* （任意）気象庁データ

---

## 🧩 レイヤー設計

### ① Presentation Layer（UI層）

**役割：表示・ユーザー操作**

* カレンダー表示
* 地域入力フォーム
* 天気アイコン表示
* ホバー詳細表示

```
src/components/
 ├ Calendar.tsx
 ├ WeatherCell.tsx
 ├ LocationInput.tsx
```

---

### ② Application Layer（フロント制御）

**役割：データ取得・状態管理**

* API呼び出し
* データ整形
* 状態管理（useState / Zustand）

```
src/hooks/
 ├ useWeather.ts

src/services/
 ├ weatherService.ts
```

---

### ③ API Layer（BFF: Backend for Frontend）

**役割：フロントと外部APIの橋渡し**

* フロント用にレスポンス整形
* APIキー隠蔽
* キャッシュ制御

```
/api/weather.ts
```

---

### ④ Backend Layer（Python）

**役割：ビジネスロジック・外部API統合**

* Open-Meteo呼び出し
* 天気コード変換
* データ統一フォーマット化

```
backend/
 ├ main.py
 ├ services/
 │   ├ weather.py
 ├ models/
 │   ├ weather_model.py
```

---

### ⑤ External Layer（外部API）

**役割：データ提供**

* Open-Meteo
* Geocoding API

---

## 🔄 データフロー

```
1. ユーザーが地域入力
2. フロントが /api/weather を呼ぶ
3. APIがPythonバックエンドを呼ぶ
4. バックエンドがOpen-Meteo APIを叩く
5. データ整形して返却
6. カレンダーに表示
```

## 🌐 API設計

### GET /api/weather

#### Request

```
/api/weather?lat=35.6&lon=140.1
```

#### Response

```json
{
  "data": [
    {
      "date": "2026-04-10",
      "temp_max": 20,
      "temp_min": 12,
      "weather": "sunny",
      "rain": 0
    }
  ]
}
```

---

## 🧠 天気コード変換

| code | 意味 |
| ---- | -- |
| 0    | 晴れ |
| 1-2  | 曇り |
| 3    | 雨  |
